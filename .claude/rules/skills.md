# 스킬 작성/운영 규칙

## 목적
- 반복 작업을 스킬로 분리해 재사용성과 품질을 높인다.

## 스킬을 만들 때
- 같은 유형의 작업이 3회 이상 반복될 때
- 절차가 길거나 체크리스트가 필요한 작업일 때
- 도메인 지식(예: Google API 연동)을 표준화해야 할 때

## 권장 구조(프로젝트 기준)
- `.claude/skills/<skill-name>/SKILL.md`
- 필요 시 하위 폴더:
- `scripts/` 자동화 스크립트
- `references/` 핵심 참고자료 요약
- `templates/` 반복 입력 템플릿

## SKILL.md 작성 원칙
- 한 스킬 = 한 책임
- 트리거 문구(언제 이 스킬을 쓰는지) 명확히 작성
- 입력/출력/실패 조건/검증 기준 포함
- 길게 쓰기보다 실행 가능한 절차 중심으로 작성

## 품질 관리
- 스킬 추가 후 최소 1회 실제 작업에 적용해 검증
- 실패/누락 패턴이 나오면 즉시 문서 갱신
- 스킬 간 중복이 커지면 병합 또는 역할 분리
