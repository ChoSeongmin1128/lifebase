# 보안/민감정보 규칙

## 금지
- 토큰, API 키, 비밀번호, 내부 주소를 문서/코드에 하드코딩 금지
- 민감정보를 커밋/PR 본문/로그에 노출 금지

## 원칙
- 민감정보는 환경변수/비밀저장소로 관리
- 샘플 문서에는 마스킹 값만 사용
- 보안 이슈 상세는 비공개 채널에서만 공유

## 사용자 격리
- 사용자 간 정보가 완전히 격리되어야 한다
- API 응답에 다른 사용자의 존재나 정보가 노출되지 않아야 한다
- 모든 API 엔드포인트에서 사용자 ID 기반 데이터 필터링을 적용한다
- 공유 맥락(공유 폴더 등) 외에는 다른 사용자의 존재를 알 수 없어야 한다
