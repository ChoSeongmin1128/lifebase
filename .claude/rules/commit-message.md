# 커밋 메시지 규칙

## 형식
```text
<type>: <한글 요약>

- 수정 내용 1
- 수정 내용 2
```

## 제목 규칙
- `<type>`은 영문 소문자
- `:` 뒤 한 칸 + 한글 요약
- 한 줄 제목, 마침표 생략

## 본문 규칙
- 본문 생략 금지, 한글로 작성
- `- ` 불릿으로 작성
- 각 항목은 \"무엇을 왜 바꿨는지\"를 포함

## type 목록
- `feat`, `fix`, `refactor`, `docs`, `test`, `chore`, `perf`, `ci`, `build`, `revert`

## 금지
- `update`, `수정`, `작업` 같은 모호한 제목
- type 없는 제목
- `Co-Authored-By` 등 자동 생성 서명 추가 금지
