# 마일스톤

## 목적
- 전체 기능을 구현하되, 의존성 순서대로 개발해 재작업을 줄인다.

## 입력
- 대상 플랫폼: Web, macOS Desktop, Windows Desktop, iOS, Android
- 대상 기능: 클라우드/공유/갤러리/동기화/캘린더/할 일/관리자
- 확정 스택: Go(chi, 헥사고날) + PostgreSQL + Redis(Asynq) + Next.js + Tauri + React Native
- 운영 조건: macOS 호스트 + Thunderbolt 외장 12TB

## 결정
- 1단계 기반
- 사용자/권한/용량 모델, Google OAuth, 헥사고날 기본 골격
- 1 API Endpoint = 1 Controller = 1 UseCase 규칙 확정
- 2단계 클라우드 코어
- 파일 CRUD, 휴지통, 관리자 용량 정책
- 3단계 공유
- 공유 폴더, 10분 초대 토큰, 수락 후 ACL
- 4단계 갤러리
- 미디어 인덱싱, 썸네일/프리뷰, 대량 리스트 성능
- 5단계 동기화 클라이언트
- 데스크톱(macOS/Windows) 로컬 폴더 동기화
- 모바일(iOS/Android) 자동 업로드/삭제 연동 옵션
- 6단계 일정/할 일 통합
- Google Calendar/Tasks 양방향 동기화 + 앱 내 연결
- 7단계 하드닝
- 관측성, 장애복구, 배포 자동화, 플랫폼별 QA

## 열린 질문
- Tauri auto-update 정책(Stable/Beta 채널 분리 여부)
- 모바일 백그라운드 업로드 실패 복구 UX 정책
- 백업 운영 시작 시점(현재는 보류)
