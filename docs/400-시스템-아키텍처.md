# 시스템 아키텍처

## 목적
- 웹/데스크톱(macOS, Windows)/모바일(iOS, Android)을 동시에 지원하는 공통 아키텍처 기준을 정의한다.

## 입력
- 단일 인증: Google OAuth
- 단일 백엔드: 계정/권한/클라우드/동기화 규칙 공통 처리
- 다중 클라이언트: Web, Desktop, Mobile

## 결정
- 서버 중심 도메인 규칙: 권한, 용량, 공유, 동기화 충돌 정책은 서버가 최종 판단
- 클라이언트 역할 분리:
- 웹: 관리/운영 + 전체 기능 접근
- 데스크톱 앱: 로컬 폴더 동기화 에이전트 포함
- 모바일 앱: 자동 업로드/갤러리 소비 중심
- 공통 API: 모든 플랫폼이 동일 API 계약 사용
- 저장소 분리: 파일 본문(스토리지) + 메타데이터(DB) 분리

## 열린 질문
- 실시간 동기화(소켓)와 주기 동기화(폴링)의 혼합 비율
- 데스크톱 앱에서 백그라운드 동기화 프로세스 운영 방식
