# 디자인 규칙

## 목적
- Lifebase 전반의 디자인 방향과 원칙을 정의한다.

## 상태
- 추후 상세 기획 예정

## 디자인 방향 (초안)
- 깔끔하고 차분한 톤 (플랫 + 미니멀)
- 정보 밀도 적절히 유지 (클라우드/캘린더에서 과도한 여백 지양)
- 레이아웃: 사이드바 네비게이션 + 메인 콘텐츠 영역
- 모바일: 사이드바 → 하단 탭바 전환

## 열린 질문
- 참고할 디자인 레퍼런스 (Notion, Linear, iCloud, Google Calendar 등)
- 정보 밀도 수준 (컴팩트 vs 여유)
- 타이포그래피 (시스템 폰트 vs 웹폰트)
