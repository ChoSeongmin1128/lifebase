# 디자인 규칙

## 목적
- Lifebase 전반의 디자인 방향과 원칙을 정의한다.

## 상태
- 프로토타입 기준 3차 규칙 반영 (2026-02-24)

## 디자인 방향 (초안)
- 깔끔하고 차분한 톤 (플랫 + 미니멀)
- 정보 밀도 적절히 유지 (클라우드/캘린더에서 과도한 여백 지양)
- 레이아웃: 사이드바 네비게이션 + 메인 콘텐츠 영역
- 탭 구조: Cloud / Calendar / Todo / Gallery / Settings 공통 탭 유지
- 데스크톱/웹 사이드바는 접기/펼치기 토글 지원 (펼침 208px / 접힘 52px 아이콘만)
- 모바일: 하단 탭바 중심
- 캘린더: 이벤트(바/텍스트)와 Todo(칩) 시각 분리 유지
- 캘린더 Year Compact: 텍스트 미노출, 마커+집계 중심 스캔 UI 유지
- 캘린더 Year Timeline: 정보 확인 UI로 유지, 중복 정보(좌측 별도 날짜열)는 제거
- 설정성 옵션(계정 필터, 색상 범례, 완료 Todo 토글, Compact/Timeline 모드)은 Settings 탭에서 통합 관리
- 운영 지표(예: 동기화 큐/충돌 건수)는 기본 숨김, 문제 발생 시 alert + 알림 목록으로 우선 노출
- 동기화 정상 상태에서는 정보량을 줄이고, 해결 필요 상태에서만 강한 시각 강조 사용
- 웹 레이아웃은 뷰포트 높이를 꽉 채우는 flex 기반 레이아웃 사용 (하단 여백 제거)
- Cloud 파일 테이블 헤더 셀 간 세로 구분선 표시 (컬럼 리사이즈 핸들 인지성 확보)
- Gallery는 앨범(기본)/리스트/상세 3가지 뷰 전환 지원, 앨범은 날짜별 그룹핑 썸네일 격자

## 사이드바
- Web/Desktop: 접기/펼치기 토글 지원
- 접힌 상태: 52px, 아이콘만 표시 (라벨 `opacity: 0`, `width: 0`)
- 펼친 상태: 208px, 아이콘 + 라벨
- 상태 저장: `localStorage('lifebase-sidebar')` (collapsed/expanded)
- 토글 버튼: 사이드바 상단 우측, 패널 레이아웃 아이콘

## 아이콘
- 인라인 SVG 사용 (Feather Icons 스타일: stroke 기반, 18×18px)
- 유니코드 이모지 사용 금지
- 활성 탭: 아이콘 stroke #fff 전환
- 아이콘 목록: Cloud(구름), Calendar(캘린더+핀), Todo(체크마크 원), Gallery(이미지 프레임), Settings(톱니바퀴)

## Settings 구조
- 카테고리별 카드 레이아웃:
  - **일반**: Google 계정 연결, 테마 모드 (라이트/다크/시스템)
  - **캘린더**: 계정 필터, 연간 보기 모드 (Compact/Timeline), 색상 범례
  - **Todo**: 완료 Todo 표시 (숨김/표시)
  - **Cloud**: 폴더 동기화 정책 (웹/데스크탑만)
- 설정 변경은 즉시 반영 (저장 버튼 불필요)

## 캘린더 화면 툴바
- 노출 항목: 뷰 전환(Year/Month/Week) + 이전/오늘/다음 + 기간 라벨
- 제거 항목: 계정 필터, 완료 Todo 토글, Compact/Timeline 스위치, 색상 범례 → Settings로 이동
