# Todo 기능

## 목적
- LifeBase Todo 앱의 기능 범위를 정의한다.

## 상태
- Year 뷰 연동 규칙 1차 반영 (2026-02-24)
- Google Tasks 유사 UI 우선 프로토타입 반영 (2026-02-24)
- Todo 시스템 설계 확정: 계층/우선순위/고정/완료/정렬/드래그 (2026-02-26)

## 기능 범위
- 리스트 중심 구조 (목록 선택 + Todo 추가 + 완료 체크)
- 하위 Todo (2단계 계층: 부모-자식-손자)
- 우선순위 4단계 (긴급/높음/보통/낮음)
- 고정(Pin) 리스트당 최대 5개
- 마감일 설정 (날짜 + 시간)
- 메모/설명 필드
- 정렬 (마감일순/우선순위순/생성일순/수동 드래그)
- 필터 (오늘/이번 주/마감 지남/우선순위별)
- 캘린더 연결 (마감일 있는 Todo → 캘린더에 표시)
- 반복 Todo (LifeBase 자체 기능)
- 완료 체크/해제 + 2초 유예 애니메이션
- 드래그앤드롭 (재정렬, 리스트 간 이동, 들여쓰기/내어쓰기)

## Google Tasks 동기화
- **동기화 안 함**: Todo는 Google Tasks와 동기화하지 않는다
- Todo의 모든 데이터는 LifeBase DB에서만 관리한다
- 계층(2단계), 우선순위, 시간, 반복, 색상, 고정 등 모든 필드가 LifeBase 전용

## 결정 (계층 구조)
- 최대 2단계 중첩 (부모-자식-손자)
- 자식 Todo는 부모 대비 좌측 24px 들여쓰기 + 세로 연결선(1px)
- 접기/펼치기: 자식이 있는 부모에 chevron 토글, 접힌 상태에서 `완료수/전체수` 배지 표시
- 기본 상태: 펼침

### 하위 Todo 생성
- 방법 1: 부모 Todo 하단 "하위 Todo 추가" Ghost 버튼 → 인라인 입력
- 방법 2: 기존 Todo를 다른 Todo 위로 드래그 후 우측 이동 → 자식으로 편입
- 방법 3: 컨텍스트 메뉴 "하위로 이동"
- 방법 4 (Web/Desktop): `Tab` 들여쓰기, `Shift+Tab` 내어쓰기

### 부모 완료 동작
- 미완료 자식이 존재하면 확인 다이얼로그: "하위 Todo N개를 함께 완료하시겠습니까?"
- "모두 완료" (primary) / "이것만 완료" (secondary)
- 자식 전체 완료 시 부모 자동 완료하지 않음 (독립적)

## 결정 (우선순위)
- 4단계: 긴급(`urgent`) / 높음(`high`) / 보통(`normal`) / 낮음(`low`)
- 시각: 깃발 SVG 아이콘 (긴급=빨간 채움, 높음=주황 채움, 보통=미표시, 낮음=회색 윤곽)
- 색상 매핑: 긴급=`error`, 높음=`warning`, 보통=없음, 낮음=`text-muted`
- 위치: 체크박스와 제목 사이
- 기본값: 새 Todo 생성 시 "보통"

## 결정 (고정 Pin)
- 고정된 Todo는 정렬 기준과 무관하게 리스트 최상단 별도 그룹에 표시
- 고정 그룹 내부에서도 현재 활성 정렬 규칙 동일 적용
- 고정 그룹과 일반 그룹 사이에 시각적 구분선(1px)
- 리스트당 최대 5개, 초과 시 토스트 안내
- 시각: `primary` 색상 핀 SVG 아이콘 (미고정 시 hover에서만 윤곽 노출)
- 고정 행 배경: `surface-accent` 색상
- 토글: 핀 아이콘 클릭 / 컨텍스트 메뉴 / 모바일 스와이프

## 결정 (완료 동작)
- 체크 → 즉시 취소선 + 뮤트 색상 적용
- 200ms: 체크 아이콘 `success` 색상 플래시
- 2초 대기: 이 시간 동안 체크 해제 가능 (실수 방지)
- 2초 후: 높이 축소 애니메이션(300ms) → 완료 섹션으로 이동 또는 숨김
- 완료 시 고정 자동 해제 (`is_pinned = false`)
- 완료 행: `text-decoration: line-through` + `opacity: 0.6` + 우선순위 아이콘 숨김

### "완료 Todo 표시" 토글
- Settings > Todo에서 설정
- 숨김(기본): 완료 Todo는 리스트에서 숨김
- 표시: 리스트 하단 "완료됨 (N)" 접이식 섹션, 완료 시각 역순 정렬, 클릭으로 미완료 복원 가능

## 결정 (정렬)

### 정렬 옵션
| 옵션 | 키 | 방향 | 설명 |
|---|---|---|---|
| 마감일순 (기본) | `due` | 오름 | 가까운 마감일 먼저, 없는 항목 최하단 |
| 우선순위순 | `priority` | 내림 | 긴급 > 높음 > 보통 > 낮음 |
| 생성일순 | `created` | 내림 | 최신 생성 먼저 |
| 수동 정렬 | `manual` | — | 사용자 드래그 순서 |

### Tie-breaker 체인
| 1차 기준 | 2차 | 3차 | 4차 |
|---|---|---|---|
| 마감일 오름 | 우선순위 내림 | 생성일 오름 | 제목 가나다 |
| 우선순위 내림 | 마감일 오름 | 생성일 오름 | 제목 가나다 |
| 생성일 내림 | 우선순위 내림 | 마감일 오름 | 제목 가나다 |

### 하위 Todo 정렬
- 하위 Todo는 부모 바로 아래에 들여쓰기 표시
- 하위 Todo끼리 독립적으로 현재 정렬 규칙 적용
- 부모 위치는 부모 자체 속성으로 결정 (하위 속성에 영향받지 않음)

### 수동 정렬과 자동 정렬 전환
- 수동 정렬 시 `sort_order` 필드 저장, 드래그로 변경
- 자동 정렬 전환 시 `sort_order` 보존 (DB에서 삭제 안 함)
- 수동 복귀 시 마지막 `sort_order` 기준 복원
- 방향 토글 제공하지 않음 (각 기준의 자연스러운 방향 고정)

### 정렬 UI
- Todo 뷰 툴바에 정렬 아이콘 버튼 + 드롭다운
- 수동 정렬 선택 시 각 Todo 좌측에 드래그 핸들(grip-vertical SVG) 표시
- 자동 정렬 모드에서는 드래그 핸들 숨김
- 하단에 "기본 정렬로 돌아가기" 옵션

### 정렬 상태 저장
- 프로토타입/0.x: `localStorage` 키 `lifebase-sort-todo`
- 1.0 이후: 서버 저장 추가, 로그인 시 동기화

## 결정 (드래그앤드롭)

### 같은 리스트 내 재정렬
- 수동 정렬 모드에서만 드래그 재정렬 활성화
- 드래그 중 반투명 고스트 + 삽입 위치에 2px `primary` 수평선
- 고정 그룹 내에서만 재정렬 가능, 고정↔일반 간 드래그로 고정 토글

### 리스트 간 이동
- Web/Desktop: Todo를 좌측 리스트 패널의 다른 리스트로 드래그
- Mobile: 롱프레스 컨텍스트 메뉴 → "다른 목록으로 이동" → 리스트 선택 하단 시트

### 들여쓰기/내어쓰기 (계층)
- 수평 우측 이동(24px 이상): 바로 위 Todo의 자식으로
- 수평 좌측 이동: 최상위로 승격
- 부모 드래그 시 자식도 함께 이동
- 2단계 초과 방지: 이미 손자인 Todo는 더 들여쓰기 불가

### 모바일 드래그
- 롱프레스(500ms) → 진동 피드백 + elevation 효과
- 들여쓰기: 수평 이동 감지
- 취소: 화면 밖 드래그 또는 다른 손가락 탭

## 결정 (리스트 구조)

### 리스트 전환 UI
- Web/Desktop: 좌측 카드 패널, 활성 리스트에 `primary` 좌측 보더 + `surface-accent` 배경, 미완료 수 배지
- Mobile: 상단 가로 칩 바로 리스트 전환 (가로 스크롤)
- 리스트 하단 "+ 새 목록" Ghost 버튼

### 기본 리스트
- 첫 번째 리스트가 기본 리스트 (삭제 불가)
- 새 Todo 생성 시 현재 활성 리스트에 추가

### 검색/필터
- Todo 탭 상단 검색 입력: 모든 리스트 대상 제목/메모 검색, 결과에 리스트명 라벨
- 필터 칩: "오늘" / "이번 주" / "마감 지남" / 우선순위 필터
- "마감 지남" 칩: 미완료 + due < 오늘, `error` 색상

## Todo 행 레이아웃
```
[drag-handle] [checkbox] [priority-flag] [title + due-badge] ... [pin-icon] [more-btn]
              |                           |-- 자식은 24px indent--|
              |-- 부모이면 chevron 토글 --|
```
- drag-handle: 16px (수동 정렬 모드 hover 시에만, grip-vertical SVG)
- checkbox: 20px 원형 (미완료=윤곽, 완료=`primary` 채움+체크)
- priority-flag: 16px (보통이면 미표시)
- title: flex-grow, 단일 행, overflow ellipsis
- due-badge: 우측 정렬, 마감 지남 시 `error` 색상
- pin-icon: 16px (고정 시 항상, 미고정 시 hover에서만)
- more-btn: 16px (more-vertical SVG → 컨텍스트 메뉴)

## 캘린더 연동 규칙
- Todo는 due(날짜) 기준으로 해당 날짜 셀에 표시한다
- Todo는 이벤트와 분리된 칩 UI로 표시한다 (이벤트 바/텍스트와 시각 구분)
- 완료 Todo는 기본 숨김이며, 사용자 토글로 표시/숨김 전환 가능
- 셀 공간이 부족할 때 숨겨진 항목(이벤트+Todo)은 `+n`으로 통합 표시
- 이벤트 우선 정책 유지 (공휴일/이벤트 텍스트 슬롯 우선, Todo는 후순위)

## DB 스키마 초안

### todo_lists
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | UUID PK | 내부 식별자 |
| user_id | UUID (INDEX) | 소유자 |
| title | VARCHAR(255) | 리스트 이름 |
| color | VARCHAR(7) | 리스트 색상 hex |
| position | INT | 리스트 표시 순서 |
| is_default | BOOLEAN | 기본 리스트 여부 |
| created_at | TIMESTAMPTZ | 생성 시각 |
| updated_at | TIMESTAMPTZ | 수정 시각 |

### todos
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | UUID PK | 내부 식별자 |
| user_id | UUID (INDEX) | 소유자 |
| list_id | UUID (INDEX) | 소속 리스트 |
| parent_id | UUID (INDEX, nullable) | 부모 Todo (null=최상위) |
| title | VARCHAR(1024) | 제목 |
| notes | TEXT | 메모 (최대 8192자) |
| due_date | DATE (nullable) | 마감일 |
| due_time | TIME (nullable) | 마감 시간 |
| status | VARCHAR(20) | `needs_action` / `completed` |
| completed_at | TIMESTAMPTZ (nullable) | 완료 시각 |
| priority | VARCHAR(10) | `urgent`/`high`/`normal`/`low` (기본: normal) |
| is_pinned | BOOLEAN | 고정 여부 (기본: false) |
| pinned_at | TIMESTAMPTZ (nullable) | 고정 시각 |
| sort_order | BIGINT | 수동 정렬 순서 |
| color | VARCHAR(7) (nullable) | 개별 Todo 색상 |
| recurrence_rule | VARCHAR(255) (nullable) | 반복 규칙 RRULE |
| deleted | BOOLEAN | 소프트 삭제 |
| created_at | TIMESTAMPTZ | 생성 시각 |
| updated_at | TIMESTAMPTZ | 수정 시각 |

### 인덱스
- `idx_todos_user_list`: (user_id, list_id)
- `idx_todos_parent`: (parent_id)
- `idx_todos_due`: (user_id, due_date)
- `idx_todos_status`: (user_id, list_id, status)

## 열린 질문
- ~~반복 Todo 생성 규칙~~ → 완료 시 다음 1개 자동 생성 (RRULE 기반 다음 날짜 계산)
- ~~캘린더 이벤트와 Todo 연결~~ → 연결 안 함. due 날짜 기반 캘린더 표시만
- ~~오프라인 Todo 큐잉~~ → 로컬 큐 + 온라인 복귀 시 순차 전송, 충돌 시 서버 우선
