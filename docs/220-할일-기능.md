# Todo 기능

## 목적
- LifeBase Todo 앱의 기능 범위를 정의한다.

## 상태
- Year 뷰 연동 규칙 1차 반영 (2026-02-24)
- Google Tasks 유사 UI 우선 프로토타입 반영 (2026-02-24)

## 기능 범위 (초안)
- Google Tasks와 유사한 리스트 중심 구조 (목록 선택 + 작업 추가 + 완료 체크)
- 완료 체크/해제
- 우선순위 (LifeBase 자체 기능, Google Tasks 미지원)
- 마감일 설정 (Google Tasks는 날짜만, 시간은 LifeBase 자체)
- 메모/설명 필드
- 정렬 (마감일순, 우선순위순, 생성일순, 수동 드래그)
- 필터 (오늘, 이번 주, 완료 숨기기/보기)
- 캘린더 연결 (마감일 있는 Todo → 캘린더에 표시)
- 반복 Todo (LifeBase 자체 기능, Google Tasks 미지원)
- Google Tasks 양방향 동기화 (title, notes, due, status, parent만)

## UI 우선순위
- 1차 프로토타입은 Google Tasks 사용 패턴(간단 입력, 체크 중심, due 중심 정렬)을 우선 재현한다.
- 우선순위/반복/시간 등 확장 기능은 기본 흐름을 해치지 않는 보조 정보로 노출한다.

## Google Tasks 동기화 범위
- 양방향: title, notes, due(날짜만), status, parent(계층)
- LifeBase 전용(동기화 안 됨): 우선순위, 시간, 반복, 색상/라벨

## 캘린더(Year 뷰) 연동 규칙
- Todo는 Google Tasks의 due(날짜) 기준으로 해당 날짜 셀에 표시한다.
- Todo는 이벤트와 분리된 칩 UI로 표시한다. (이벤트 바/텍스트와 시각 구분)
- 완료 Todo는 기본 숨김이며, 사용자 토글로 표시/숨김 전환 가능해야 한다.
- 셀 공간이 부족할 때 숨겨진 항목(이벤트+Todo)은 `+n`으로 통합 표시한다.
- 이벤트 우선 정책을 유지한다. (공휴일/이벤트 텍스트 슬롯 우선, Todo는 후순위)

## 열린 질문
- 우선순위 단계 (높음/중간/낮음 3단 vs 1~4 숫자)
- 캘린더와 Todo의 관계 (명확 분리 vs 느슨한 통합)
- 반복 Todo의 생성 규칙 (완료 시 다음 생성? 미리 생성?)
