# Todo 기능

## 목적
- LifeBase Todo를 Google Tasks 연동 전제로 재정의한다.

## 상태
- Year 뷰 연동 규칙 1차 반영 (2026-02-24)
- Google Tasks 유사 UI 프로토타입 반영 (2026-02-24)
- Todo 정책 재정의: 다중 계정 + Google Tasks 원본 중심 (2026-02-27)
- 완료 항목 동기화/보존 기간 설정/출처 표기 확장 반영 (2026-03-05)
- Web Todo 인라인 편집 UX를 Google Tasks식 확장 행 중심으로 재정리 (2026-03-09)

## 제품 원칙 (2026-02-27 확정)
- Todo의 단일 원본(Source of Truth)은 Google Tasks다.
- LifeBase는 동기화 캐시/검색/통합 UI 역할을 수행한다.
- UI 디자인 언어는 기존 LifeBase 문서(색/타이포/컴포넌트 규칙)를 유지한다.
- 기능은 Google Tasks 호환 범위를 우선한다.

## 기능 범위 (1차)
- 목록 중심 구조 (Task List 선택 + Todo 생성/수정/완료)
- 계층 구조: 1단계 하위 Todo만 허용 (부모-자식)
- 마감일(due date) 기반 표시 및 캘린더 연동
- 수동 정렬(같은 계정 + 같은 Task List 내부에서만)
- 다중 Google 계정 동시 연결 및 계정/리스트 필터
- 통합 Inbox 뷰 (여러 계정/리스트를 한 화면에서 조회)

## 비범위 (1차)
- 우선순위(priority)
- 고정(Pin)
- 반복 Todo
- 마감 시간(due time)
- 색상/태그/라벨 확장
- 계정 간 직접 드래그 이동

## Google Tasks 동기화
- Todo 생성/수정/삭제/완료는 Google Tasks API에 우선 반영한다.
- 로컬 DB는 `google_account_id`, `google_tasklist_id`, `google_task_id` 기준 캐시를 저장한다.
- 폴링은 계정별로 독립 실행하며 `updatedMin` 커서를 별도 관리한다.
- 동기화 장애는 계정 단위로 격리한다. (A 계정 오류가 B 계정 동기화를 막지 않음)
- 로컬 삭제가 push outbox에 대기 중인 Todo와 그 직계 자식은 background pull sync가 다시 복구하지 않는다.
- `hidden=true` 상태라도 완료 항목(`status=completed` 또는 `completed!=null`)은 동기화 대상에 포함한다.
- 완료 보존 기간은 사용자 설정 `todo_done_retention_period`로 제어한다. (`1m|3m|6m|1y|3y|unlimited`, 기본 `1y`)
- `unlimited` 설정 시 완료 항목 자동 정리(삭제) 작업을 수행하지 않는다.
- parent 변경과 수동 정렬은 Google `tasks.move`로 반영한다.
- Google Tasks에서 1단계를 넘는 parent 체인이 들어오면 LifeBase는 최상위 부모 기준 1단계로 정규화한다.

## 결정 (계층 구조)
- 최대 1단계 중첩만 허용한다.
- 들여쓰기는 `24px` 기준을 유지한다.
- 자식 Todo 생성 경로: 행 컨텍스트 메뉴 "하위 Todo 추가", 드래그 우측 이동, `Tab`/`Shift+Tab` (Web/Desktop)
- 완료된 자식 Todo는 부모와 다른 완료 상태여도 부모 아래 계층을 유지해 표시한다.

## 결정 (정렬/이동)
- Todo 정렬 옵션은 `manual`, `due`, `recent_starred`, `title` 4종으로 고정한다.
- 기본 정렬은 `due`다.
- 수동 정렬은 Google `tasks.move` 기준으로 반영한다.
- 드래그 재정렬은 같은 `google_account_id + google_tasklist_id` 내부에서만 허용한다.
- 다른 리스트/계정으로 이동은 `copy + delete` 방식으로 처리한다.
- 검색/필터 활성화 상태에서는 드래그를 비활성화한다.

## 결정 (삭제 UX)
- 단건 삭제와 완료 항목 일괄 삭제는 즉시 영구 반영하지 않고 우측 하단 Undo 토스트 5초를 먼저 노출한다.
- 사용자가 5초 안에 `실행 취소`를 누르지 않으면 실제 삭제 API를 호출한다.
- 목록 삭제처럼 더 파괴적인 액션은 확인 다이얼로그를 유지한다.

## 결정 (다중 계정 UX)
- 모든 Todo 행에 계정 배지와 리스트 라벨을 표시한다.
- "새 Todo" 입력창은 현재 쓰기 대상(계정/리스트)을 항상 표시한다.
- 기본 쓰기 대상은 Settings에서 지정한다.
- 통합 뷰에서 편집 시 원본 계정/리스트를 유지한다.
- 재인증 필요 계정은 읽기 전용으로 전환하고 재연결 CTA를 노출한다.
- 목록 출처 라벨은 `로컬`, `Google · {계정이메일}`, `Google · 계정 미확인`, `통합`으로 고정한다.

## Todo 행 레이아웃
```
[drag-handle] [checkbox] [title + due-badge] ... [account-badge] [more-btn]
              |                 |-- 자식은 24px indent --|
              |-- 부모이면 chevron 토글 --|
```
- drag-handle: 16px (수동 정렬 모드에서만)
- checkbox: 20px 원형
- title: flex-grow, 기본 3줄까지 노출, 선택 시 같은 행에서 직접 편집
- due-badge: 우측 정렬, overdue 시 `error` 색상
- account-badge: `개인/회사/가족` 등 계정 라벨
- more-btn: 16px (more-vertical)
- expanded-details: 같은 surface 아래 메모/일정/우선순위만 보조 편집, 별도 모달/사이드패널 미사용
- expanded-motion: 높이 + 투명도 전환으로 짧고 부드럽게 열리고 닫힘
- row-switch: 다른 Todo 행 클릭은 현재 편집을 먼저 강제 종료하지 않고, 행 간 편집 전환으로 처리
- expanded-close: 같은 행 재클릭, 빈 영역 클릭, `Esc`로 닫기
- due-edit: 날짜는 Todo 내부 커스텀 달력 팝오버에서 고르고, 시간은 같은 팝오버 안에서 30분 단위 목록 선택 또는 정확한 직접 입력으로 조정하며, 내부 레이어를 조작하는 동안 행 편집은 닫히지 않음
- priority-edit: 우선순위는 단일 칩으로 현재 값만 노출하고, 클릭 시 목록 팝오버로 변경

## 캘린더 연동 규칙
- Todo는 due(날짜) 기준으로 해당 날짜 셀에 표시한다.
- 계정 필터는 캘린더 이벤트와 Todo에 동일 적용한다.
- 완료 Todo는 기본 숨김이며 사용자 토글로 표시/숨김 전환한다.
- 셀 공간 부족 시 `+n`으로 통합 표기한다.

## DB 저장 모델 (캐시 중심)

### todo_lists
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | UUID PK | 내부 식별자 |
| user_id | UUID (INDEX) | 소유자 |
| google_account_id | UUID (INDEX) | 연결 Google 계정 |
| google_id | TEXT (INDEX) | Google Task List ID |
| name | VARCHAR(255) | 리스트 이름 |
| sort_order | INT | 표시 순서 |
| last_synced_at | TIMESTAMPTZ | 마지막 동기화 시각 |
| created_at | TIMESTAMPTZ | 생성 시각 |
| updated_at | TIMESTAMPTZ | 수정 시각 |

### todos
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | UUID PK | 내부 식별자 |
| user_id | UUID (INDEX) | 소유자 |
| list_id | UUID (INDEX) | 내부 리스트 FK |
| google_account_id | UUID (INDEX) | 연결 Google 계정 |
| google_id | TEXT (INDEX) | Google Task ID |
| parent_id | UUID (INDEX, nullable) | 부모 Todo (null=루트) |
| title | VARCHAR(1024) | 제목 |
| notes | TEXT | 메모 |
| due | DATE (nullable) | 마감일 |
| status | VARCHAR(20) | `needsAction` / `completed` |
| sort_order | INT | 형제 내 수동 정렬 순서 |
| deleted_at | TIMESTAMPTZ (nullable) | 소프트 삭제 |
| last_synced_at | TIMESTAMPTZ | 마지막 동기화 시각 |
| created_at | TIMESTAMPTZ | 생성 시각 |
| updated_at | TIMESTAMPTZ | 수정 시각 |

### 인덱스
- `idx_todos_user_account_list`: (user_id, google_account_id, list_id)
- `idx_todos_google_key`: (google_account_id, google_id)
- `idx_todos_parent`: (parent_id)
- `idx_todos_due`: (user_id, due)
