# 메모 기능

## 목적
- Apple Memo 수준의 빠른 캡처 UX와 Markdown 중심 작성 경험을 결합한 Memo 기능 범위를 정의한다.

## 상태
- 1차 스펙 초안 정의 (2026-03-03)

## 기능 범위 (1차)
- 노트 생성/수정/삭제
- 제목 + 본문(`content_md`) 저장
- 검색(제목/본문)
- 고정(Pin) 정렬
- Markdown 기반 편집 + 즉시 렌더
- 코드 블록(언어 지정 fenced block) 지원

## 결정 (편집 원칙)
- 저장 포맷은 Markdown 원문을 단일 소스(`content_md`)로 사용한다.
- 편집 UX는 WYSIWYG 느낌을 유지하되 Markdown 키 입력을 우선 지원한다.
- Notion 전체 블록 시스템(DB/협업/실시간 공동 편집)은 1차 범위에서 제외한다.

## 결정 (자동 변환 규칙)
- `->` 입력 시 `→` 자동 변환
- `---` + Enter 입력 시 구분선 변환
- `# `, `## `, `### ` + Space 입력 시 헤딩 변환
- `* `, `- `, `1. ` + Space 입력 시 리스트 변환
- `[] `, `[x] ` + Space 입력 시 체크리스트 변환
- ````` + Enter 입력 시 코드 블록 시작
- fenced code block 언어 지정(````ts`, ````js`) 지원

## 결정 (키보드 동작)
- `Tab`: 리스트/체크리스트 들여쓰기
- `Shift+Tab`: 리스트/체크리스트 내어쓰기
- `Enter`: 현재 블록 유형 유지한 다음 줄 생성
- 빈 리스트 항목에서 `Enter`: 리스트 종료
- 코드 블록 내부: 자동 변환 규칙 비활성

## 결정 (안정성/사용성)
- 자동 변환은 문장 시작 패턴에서만 적용한다.
- 한글 IME 조합 중에는 자동 변환을 적용하지 않고 조합 완료 시점에만 적용한다.
- 자동 변환 동작도 Undo/Redo 한 단계로 자연스럽게 복원 가능해야 한다.

## 1차 제외 범위
- Notion식 블록 DB/토글/콜아웃/임베드
- 멘션/백링크
- 다중 사용자 실시간 협업 편집

## 열린 질문
- Memo 분류 체계를 폴더 기반으로 시작할지 태그 기반으로 시작할지
- Memo와 Todo/Calendar 간 변환 액션을 1차에 포함할지 2차로 미룰지
