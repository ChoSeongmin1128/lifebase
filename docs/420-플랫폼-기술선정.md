# 플랫폼 기술선정

## 목적
- Web, macOS, Windows, iOS, Android를 어떤 기술로 구현할지 사전에 고정한다.

## 확정 기준
- 초기 사용자 수는 크지 않다.
- 로컬 연결 스토리지는 12TB를 사용한다.
- 사용자별 기본 용량 정책은 1TB 단위 할당을 기준으로 한다.
- 초기 단계부터 확장성/경량성을 우선한다.
- 서비스가 기능 대비 무겁지 않아야 한다.
- 유지보수가 쉬운 구조를 우선한다.
- 호스팅은 Mac Mini(macOS, bare metal) + Thunderbolt 연결 외장 스토리지 기준으로 운영한다.
- 배포 방식은 bare metal (Docker 미사용).
- DB 마이그레이션 도구는 goose를 사용한다.
- 파일시스템은 APFS를 사용한다 (외장 HDD 포함).
- 도메인은 `lifebase.cc` (Cloudflare Registrar, DNS 전용 모드)로 운영한다.
- 백업은 초기에는 적용하지 않고 추후 운영 단계에서 도입한다.

## 최종 선택(확정)
- 모노레포: `pnpm workspace + Turborepo`
- 공통 언어: Go(서버) + TypeScript(클라이언트)
- 웹: Next.js(App Router) + React
- 데스크톱(macOS/Windows): Tauri v2 + React(웹 UI 재사용) + 동기화 사이드카/워커
- 모바일(iOS/Android): React Native(Expo bare workflow)
- 백엔드: Go + chi + PostgreSQL + Redis(Asynq)
- 파일 저장소: 로컬 파일시스템 + DB 메타데이터 인덱스
- API 통신: REST + WebSocket
- 서버 아키텍처: 헥사고날(Ports & Adapters)
- 컴포넌트 라이브러리: shadcn/ui (Radix 기반 + Tailwind CSS)
- API 구현 규칙: 1 API Endpoint = 1 Controller = 1 UseCase
- 공통 패키지:
- `packages/domain` 도메인 모델/유효성
- `packages/api-types` API 타입 계약
- `packages/features/<feature>` 기능 공유 패키지 (현재 `todo` 적용)

## 선정 이유
- 경량성: Electron 대비 Tauri가 앱 크기/메모리 사용량에서 유리
- 확장성: Go + 헥사고날 구조로 도메인/어댑터 분리가 명확
- 유지보수: 1 API/1 Controller/1 UseCase 규칙으로 변경 영향 범위가 작음
- 운영 현실 적합성: mac 호스팅 + Thunderbolt 외장 12TB 환경에 맞춤

## 플랫폼별 작업 방식

### Web
- 역할: 전체 기능 접근, 관리자 페이지, 운영용 설정 화면
- 우선 구현: 파일관리/공유/갤러리/캘린더/Todo 핵심 화면

### Web 렌더링 안정성 규칙 (Next.js)
- 기본 원칙: 서버 렌더(SSR)와 클라이언트 첫 렌더(Hydration)의 DOM 구조/속성 동일성 유지
- 금지 패턴: `localStorage`, `matchMedia`, `Date.now()`, `Math.random()`, `typeof window` 분기에 따라 첫 렌더의 엘리먼트 구조 또는 속성(`id`, `aria-*`)을 변경하는 구현
- UI 라이브러리 주의: Radix 기반 컴포넌트(`DropdownMenuTrigger` 등)는 내부 `id`를 생성하므로, 초기 렌더 트리가 달라지면 `radix-*` id mismatch 발생 가능
- 관리자 웹 사례: 테마 드롭다운 초기 렌더 불일치 시 `A tree hydrated but some attributes...` 경고와 함께 trigger 버튼 `id` 불일치 발생 가능
- 대응 원칙 1: 서버에서 확정 불가한 값(브라우저 저장소/사용자 환경)은 첫 렌더에 직접 반영하지 않고 mount 이후 적용
- 대응 원칙 2: 경로/권한 기반 분기(`usePathname` 포함)는 서버/클라이언트에서 동일한 초기 결과를 보장하도록 설계
- 대응 원칙 3: 불가피한 경우에만 부분 `dynamic(..., { ssr: false })` 적용

### Desktop (macOS/Windows)
- 역할: 로컬 폴더 동기화의 주 클라이언트
- 구조:
- Renderer: React UI
- Tauri Core: 윈도우/권한/파일시스템 브리지
- Sync Worker: 동기화 작업 스케줄링/파일 감시/watch/충돌 처리 분리 실행
- 주의: 절전/재부팅/네트워크 단절 복구 로직 필수

### Mobile (iOS/Android)
- 역할: 자동 업로드 + 갤러리 소비 + 기본 파일 작업
- 구조:
- 앱 포그라운드: 탐색/재생/업로드 큐 확인
- 백그라운드: 사진 자동 업로드 워커
- 정책:
- \"폰에서 삭제 시 클라우드 삭제\" 기본값 OFF
- ON 선택 시에도 휴지통/유예기간 정책 적용

## 공통 비기능 요구사항
- 오프라인/재시도: 네트워크 불안정 시 큐 기반 복구
- 대용량 미디어: 청크 업로드 + 썸네일 비동기 생성
- 관측성: 플랫폼별 에러 리포트 + 서버 통합 로그 추적
