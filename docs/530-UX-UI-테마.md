# UX/UI 테마

## 목적
- 버튼, 간격, 모서리, 폰트 등 UI 컴포넌트의 통일 규칙을 정의한다.

## 상태
- 프로토타입 3차 기준 반영 (2026-02-24)

## 버튼 체계 (초안)
- Primary: 브랜드 컬러 배경, 흰 텍스트 (주요 액션: 저장, 생성)
- Secondary: 회색 배경 또는 아웃라인 (보조 액션: 취소, 닫기)
- Danger: 빨간 계열 (삭제, 해제)
- Ghost: 배경 없음, 텍스트만 (덜 중요한 액션)
- 한 화면에 Primary 버튼은 1개만

## 간격/크기 (초안)
- 4px 또는 8px 배수 그리드
- 모서리: 8px (rounded-lg) 통일
- 캘린더 상단 툴바는 핵심 탐색(뷰 전환/이동/기간 라벨)만 노출, 설정성 옵션은 Settings 탭으로 분리
- Year Timeline 날짜 라벨은 줄바꿈 없이 단일 라인으로 유지
- 사이드바: 펼침 208px / 접힘 52px, CSS transition 0.2s

## 상태/알림 컴포넌트
- 동기화 관련 운영 지표(큐, 충돌 건수)는 기본 화면에서 숨기고 고급 정보로 분리
- 가벼운 성공/정보/경고는 우측 하단 토스트로 즉시 노출
- 화면 맥락에 강하게 묶인 오류(동기화 충돌, 재인증 필요, 권한 오류)는 상단/인라인 alert 배너로 노출
- 동일 이벤트를 알림 목록(Notification Center)에 남겨 재확인 가능하게 유지
- 해결 액션 버튼(`충돌 해결`, `재시도`, `재인증`)은 alert/알림 항목에 직접 배치

## In-App 알림 세부 규칙
- 금지: 브라우저 기본 `alert/confirm/prompt` 사용 금지
- 기본 위치: 토스트는 우측 하단 고정 스택(모바일은 하단 중앙)
- 토스트 지속시간:
  - 성공/정보: 2.5~4초
  - 경고: 4~6초
  - 오류: 자동 닫힘 없이 수동 닫기 또는 8초 이상 유지
- 액션:
  - 복구 가능한 이벤트는 토스트에 단일 액션(`실행 취소`, `다시 시도`) 제공
  - 다단계/파괴적 액션은 토스트가 아니라 모달 확인 다이얼로그 사용
- 중복 억제:
  - 동일 코드/메시지 토스트는 짧은 시간 내 1회만 노출(디바운스/중복 제거)
  - 반복 실패는 카운트 업 형태로 합산 표기 (`업로드 실패 3건`)
- 알림센터 연계:
  - 토스트는 일시적 전달 채널
  - 경고/오류는 알림센터에 누적 기록하고 해결 여부 상태를 저장

## 테마 모드
- 라이트/다크/시스템 3가지 모드 선택 제공
- 기본값은 시스템
- 선택값은 사용자 로컬 설정에 저장하고 Web/Desktop/Mobile 프로토타입에서 동일 방식으로 동작

## 아이콘
- 모든 아이콘/상태 심볼/네비게이션 심볼은 SVG만 사용한다. (인라인 SVG 또는 SVG 스프라이트)
- 유니코드 이모지 사용 금지
- 유니코드 기호를 아이콘 대체로 사용하는 방식도 금지한다. (예: `↻`, `✓`, `⚠`, `▶`, `◂`, `▸`)
- 사이드바 펼침 상태: 아이콘 + 텍스트 병행
- 사이드바 접힘 상태: 아이콘만 표시
- 활성 탭: 아이콘 stroke #fff로 전환
