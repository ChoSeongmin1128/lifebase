# 핵심 사용자 플로우

## 목적
- MVP 검증에 필요한 대표 사용자 흐름만 정의한다.

## 플로우 A: 첫 연결
1. 사용자가 회원가입/로그인
2. Google 계정 연결(OAuth)
3. 동기화 범위 선택(캘린더/할 일)
4. 초기 동기화 완료 상태 확인

## 플로우 B: 일정 생성
1. 캘린더에서 일정 생성
2. 반복/알림/색상 설정
3. 저장 시 Google Calendar 반영
4. 동기화 상태 배지 표시

## 플로우 C: 할 일 생성 및 일정 연결
1. 할 일 생성
2. 마감일/우선순위 설정
3. 기존 일정과 연결
4. Google Tasks 동기화 결과 확인

## 플로우 D: 파일 업로드
1. 클라우드 화면 진입
2. 로컬 파일 업로드
3. 폴더 이동/정렬
4. 활동 로그에 기록
