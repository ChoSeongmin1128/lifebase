# 핵심 사용자 플로우

## 목적
- 전체 기능 구현 전제에서 우선순위가 높은 대표 사용자 흐름을 정의한다.

## 플로우 A: 첫 연결
1. 사용자가 회원가입/로그인
2. Google 계정 연결(OAuth)
3. 동기화 범위 선택(캘린더/Todo)
4. 초기 동기화 완료 상태 확인
5. 필요 시 추가 Google 계정 연결
6. 캘린더 전역 설정에서 계정 필터 기본값 설정 (전체/선택 계정)
7. Home으로 진입해 오늘 기준 통합 요약 확인

## 플로우 B: 일정 생성
1. 캘린더에서 일정 생성
2. 반복/알림/색상 설정
3. 저장 시 Google Calendar 반영
4. 동기화 상태 배지 표시

## 플로우 C: Todo 생성 및 일정 연결
1. Todo 생성
2. 마감일/고정 설정
3. 기존 일정과 연결
4. Google Tasks 동기화 결과 확인
5. 연간(Year) 뷰에서 Todo 마감일 표시 확인
6. 필요 시 `완료 Todo 표시` 토글로 완료 항목 포함 여부 전환
7. 필요 시 캘린더 전역 `계정 필터`로 특정 Google 계정 일정/Todo만 표시

## 플로우 D: 파일 업로드
1. 클라우드 화면 진입
2. 로컬 파일 업로드
3. 폴더 이동/정렬
4. 활동 로그에 기록

## 플로우 E: 동기화 이슈 대응
1. 사용자는 기본 화면에서 `동기화 상태`와 `마지막 동기화 시각`만 확인
2. 충돌/오류 발생 시 토스트(가벼운 안내) 또는 alert 배너(해결 필요 이슈)로 즉시 인지
3. 알림 목록에서 문제 항목(충돌, 재인증 필요, 동기화 실패) 확인
4. `충돌 해결` 또는 `재시도` 액션으로 해결 화면 진입
5. 해결 완료 시 alert 해제, 알림 목록 상태를 해결됨으로 변경

## 플로우 F: Home 허브 기반 일일 운영
1. 좌측 상단 `LB` 로고 또는 Home 탭으로 진입
2. 오늘 일정/지연 Todo/최근 파일/저장공간 요약을 한 화면에서 확인
3. Home 빠른 액션(일정 추가/Todo 추가/파일 업로드)으로 작업 시작
4. 세부 작업이 필요한 경우 각 탭(캘린더/클라우드/Todo)으로 이동
