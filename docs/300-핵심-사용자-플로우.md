# 핵심 사용자 플로우

## 목적
- 전체 기능 구현 전제에서 우선순위가 높은 대표 사용자 흐름을 정의한다.

## 플로우 A: 첫 연결
1. 사용자가 회원가입/로그인
2. Google 계정 연결(OAuth)
3. 동기화 범위 선택(캘린더/할 일)
4. 초기 동기화 완료 상태 확인
5. 필요 시 추가 Google 계정 연결
6. 캘린더 전역 설정에서 계정 필터 기본값 설정 (전체/선택 계정)

## 플로우 B: 일정 생성
1. 캘린더에서 일정 생성
2. 반복/알림/색상 설정
3. 저장 시 Google Calendar 반영
4. 동기화 상태 배지 표시

## 플로우 C: 할 일 생성 및 일정 연결
1. 할 일 생성
2. 마감일/우선순위 설정
3. 기존 일정과 연결
4. Google Tasks 동기화 결과 확인
5. 연간(Year) 뷰에서 Todo 마감일 표시 확인
6. 필요 시 `완료 Todo 표시` 토글로 완료 항목 포함 여부 전환
7. 필요 시 캘린더 전역 `계정 필터`로 특정 Google 계정 일정/Todo만 표시

## 플로우 D: 파일 업로드
1. 클라우드 화면 진입
2. 로컬 파일 업로드
3. 폴더 이동/정렬
4. 활동 로그에 기록
