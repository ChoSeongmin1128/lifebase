# 플랫폼 통일성

## 목적
- Web, Desktop, Mobile 간 디자인/UX 통일 규칙을 정의한다.

## 상태
- 프로토타입 3차 기준 반영 (2026-02-24)

## 통일 전략 (초안)
- Web + Desktop(Tauri): 동일 React 컴포넌트 공유 → 자연스럽게 통일
- Mobile(React Native): 같은 디자인 토큰 공유, 네이티브 컴포넌트 사용
- 디자인 토큰(색상, 간격, 폰트 크기 등)을 공통 패키지로 관리

## 디자인 토큰 패키지
- packages/design-tokens (또는 packages/ui-tokens)
- 색상, 간격, 폰트 크기, 모서리, 그림자 등
- CSS 변수 + JS 객체 양쪽 출력

## 플랫폼별 차이 허용 범위
- 네비게이션: 웹/데스크톱 사이드바(접기/펼치기 토글) → 모바일 하단 탭바
- 인터랙션: 웹 호버 → 모바일 롱프레스
- 그 외 시각적 요소는 최대한 통일
- 아이콘: 세 플랫폼 모두 인라인 SVG (Feather 스타일) 사용 통일
- 캘린더 Year 규칙: 세 플랫폼 모두 `Compact(마커 중심)` / `Timeline(상세 확인)` 역할 분리를 공통 적용
- 캘린더 Timeline 표기: 좌측 별도 날짜열 대신 셀 내부 날짜 라벨을 공통 사용
- 설정 구조: 세 플랫폼 모두 Settings 탭에서 카테고리별(일반/캘린더/Todo/Cloud) 통합 관리
- 캘린더 화면 툴바: 세 플랫폼 모두 뷰 전환 + 이전/오늘/다음만 노출 (설정성 옵션은 Settings로 통일)
- 동기화 이슈 노출: 세 플랫폼 모두 `alert + 알림 목록` 정책 유지 (운영 지표 기본 숨김)
- 테마 전환: 세 플랫폼 모두 라이트/다크/시스템 3모드 동일 제공
- Gallery 뷰 모드: 세 플랫폼 모두 앨범(기본)/리스트/상세 3가지 뷰 전환 동일 제공
- Year Compact 요일 헤더: 세 플랫폼 모두 각 월 상단에 요일(일~토) 표시, 날짜를 실제 요일 위치에 정렬

## 모바일 Year Timeline 규칙
- `table-layout: fixed` 필수 (열 너비 균등 고정)
- `border-collapse: separate` + `border-spacing: 0` (sticky 헤더 호환)
- 월 헤더 `th`: `position: sticky; top: 0; z-index: 10` (스크롤 시 고정)
- 타임라인 활성 시 외부 스크롤 차단, 내부 스크롤만 사용 (이중 스크롤 방지)
- `min-width: 960px` (12개월 열이 넉넉하게 표시)
