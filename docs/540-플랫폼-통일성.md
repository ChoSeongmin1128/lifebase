# 플랫폼 통일성

## 목적
- Web, Desktop, Mobile 간 디자인/UX 통일 규칙을 정의한다.

## 상태
- 프로토타입 4차 기준 반영 (2026-02-24)
- 프로토타입 5차: 캘린더 UX 개선 3플랫폼 동기화, 플랫폼별 차이 명시 (2026-02-25)

## 통일 전략 (초안)
- Web + Desktop(Tauri): 동일 React 컴포넌트 공유 → 자연스럽게 통일
- Mobile(React Native): 같은 디자인 토큰 공유, 네이티브 컴포넌트 사용
- 디자인 토큰(색상, 간격, 폰트 크기 등)을 공통 패키지로 관리

## 디자인 토큰 패키지
- packages/design-tokens (또는 packages/ui-tokens)
- 색상, 간격, 폰트 크기, 모서리, 그림자 등
- CSS 변수 + JS 객체 양쪽 출력

## 플랫폼별 차이 허용 범위
- 네비게이션: 웹/데스크톱 사이드바(접기/펼치기 토글) → 모바일 하단 탭바
- 인터랙션: 웹 호버 → 모바일 롱프레스
- 그 외 시각적 요소는 최대한 통일
- 아이콘: 세 플랫폼 모두 SVG만 사용 통일 (인라인 SVG 또는 SVG 스프라이트, 유니코드 이모지/기호 아이콘 금지)
- 캘린더 Year 규칙: 세 플랫폼 모두 `Compact(마커 중심)` / `Timeline(상세 확인)` 역할 분리를 공통 적용
- 캘린더 Timeline 표기: 좌측 별도 날짜열 대신 셀 내부 날짜 라벨을 공통 사용
- 설정 구조: 세 플랫폼 모두 Settings 탭에서 카테고리별(일반/캘린더/Todo/알림/Cloud) 통합 관리
- 캘린더 화면 툴바: 세 플랫폼 모두 단일 드롭다운(Year·Compact/Year·Timeline/Month/Week/Agenda) + chevron 이전/다음 + 오늘 텍스트 (설정성 옵션은 Settings로 통일)
- 캘린더 뷰 5종: Year·Compact / Year·Timeline / Month / Week(시간 그리드) / Agenda(일별 아젠다 리스트) — 3플랫폼 공통
- 모바일 앱 월간 뷰: 전체 월 그리드(5~6주) 동적 렌더링 (하드코딩 아님)
- 모바일 앱 주간(Week) 뷰: 시간 그리드 기반 (웹/데스크톱과 동일, 가로 스크롤 허용)
- Agenda 뷰: 일별 아젠다 리스트(날짜 헤더 + 종일/시간 이벤트 + Todo), 무한 스크롤 — 3플랫폼 공통
- 동기화 이슈 노출: 세 플랫폼 모두 `alert + 알림 목록` 정책 유지 (운영 지표 기본 숨김)
- 테마 전환: 세 플랫폼 모두 라이트/다크/시스템 3모드 동일 제공
- Gallery 뷰 모드: 세 플랫폼 모두 앨범(기본)/리스트/상세 3가지 뷰 전환 동일 제공
- Year Compact 요일 헤더: 세 플랫폼 모두 각 월 상단에 요일(일~토) 표시, 날짜를 실제 요일 위치에 정렬
- 타이포그래피: 세 플랫폼 모두 Pretendard Variable (CDN dynamic subset) 사용
- 오늘 버튼: 세 플랫폼 모두 pill 형태(`.cal-today-btn`, `border-radius: 999px`) 통일
- 드롭다운 메뉴: 플랫 리스트(연간·달력 / 연간·타임라인 / 구분선 / 월간 / 주간 / 일정) + SVG 아이콘 + 한글 라벨 + 체크마크 통일
- 드롭다운 버튼: 뷰별 동적 SVG 아이콘 + 한글 라벨 (3플랫폼 공통)
- Agenda 뷰: 일정 없는 날짜 숨김, 빈 상태 메시지 표시 (3플랫폼 공통)
- Week 시간 설정: Settings에서 시작/종료 시간 변경 가능 (3플랫폼 공통)
- Week 30분 spacer: 시작 시간 전 반 행, dashed 약한 선, 라벨 없음 (3플랫폼 공통)
- Week 이벤트 시간 표시: 시작-종료 범위 표시 (3플랫폼 공통)
- Week 겹치는 이벤트: 열 분할 나란히 배치 (3플랫폼 공통)
- Month 스패닝 바: 웹/데스크탑만 grid-column 기반 연속 바, 모바일은 단순 chip 유지
- Week all-day 스패닝 바: 웹/데스크탑만 grid-column 기반 연속 바, 모바일은 단순 chip 유지
- 연간 Compact 빈 칸: 투명 배경/테두리 (웹/데스크탑, 모바일은 원래 빈 칸 없음)
- Cloud breadcrumb: 세 플랫폼 모두 루트에서는 미표시, 하위 폴더 진입 시에만 경로 노출

## Cloud 동기화 폴더 플랫폼별 차이
- 동기화 배지(syncing/done/error): 세 플랫폼 모두 파일 목록에서 표시 (모두 SVG 아이콘 사용)
- syncing 배지: 회전 애니메이션(`animation: spin`) 포함, 세 플랫폼 모두 동일
- "이 폴더 동기화" 컨텍스트 메뉴: 데스크톱 앱 전용 (웹/모바일 미노출)
- Settings > Cloud 동기화 카드: 세 플랫폼 모두 표시, 단 데스크톱만 전체 기능(경로 변경/폴더 추가/해제)
- 웹/모바일 Settings > Cloud: 읽기 전용 (상태 확인만, "동기화 설정은 데스크톱 앱에서 관리" 안내)
- 모바일 카드: 로컬 경로 미표시 (클라우드 경로만), 액션 버튼 없음
- 웹/모바일은 클라우드 스토리지 뷰어+업로더, 데스크톱만 동기화 클라이언트 역할

## 모바일 Year Timeline 규칙
- `table-layout: fixed` 필수 (열 너비 균등 고정)
- `border-collapse: separate` + `border-spacing: 0` (sticky 헤더 호환)
- 월 헤더 `th`: `position: sticky; top: 0; z-index: 10` (스크롤 시 고정)
- 타임라인 활성 시 외부 스크롤 차단, 내부 스크롤만 사용 (이중 스크롤 방지)
- `min-width: 960px` (12개월 열이 넉넉하게 표시)

## 정렬 규칙 (플랫폼 공통)
- 캘린더 정렬: 시스템 고정(시작일 → 기간 → 제목), 사용자 옵션 없음 — 3플랫폼 공통
- 캘린더 lane 배정: 시작일 오름차순 → 기간 내림차순 → 제목 가나다순 — 3플랫폼 공통
- Todo 정렬 옵션: 마감일/우선순위/생성일/수동 — 3플랫폼 공통
- Todo 기본 정렬: 마감일 오름차순 — 3플랫폼 공통
- Cloud 기본 정렬: 이름 오름차순, 폴더 항상 우선 — 3플랫폼 공통
- Gallery 기본 정렬: 수정한 날짜 내림차순 — 3플랫폼 공통
- 정렬 상태 저장: `localStorage` (0.x), 서버 동기화 (1.0+) — 3플랫폼 공통
- "기본 정렬로 돌아가기" 옵션: 정렬 드롭다운 하단 — 3플랫폼 공통

## 알림 규칙 (플랫폼 공통)
- In-App 알림(토스트/배지): 5플랫폼 공통
- Push 알림: Web(FCM Web Push), Desktop macOS/Windows(Electron 알림), iOS(APNs), Android(FCM)
- DND 시간대 스케줄: Settings에서 설정 — 5플랫폼 공통
- 이벤트 알림 최대 5개: Google Calendar 제한과 동일 — 5플랫폼 공통
- Email 알림: 1.x에서 추가 예정
