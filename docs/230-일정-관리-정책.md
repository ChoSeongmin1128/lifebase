# 일정 관리 정책

## 목적
- 중복 일정, 충돌, 동기화 시 데이터 관리 정책을 정의한다.

## 상태
- 추후 상세 기획 예정

## 중복 일정 관리
- 같은 시간대 겹침: 중복 방지 강제 안 함 (실생활에서 일정은 겹칠 수 있음)
- 시각적으로 겹침 표현 (나란히 배치 또는 겹쳐서 표시)
- Google 동기화 중복: google_id + etag 기반 동일 일정 판별, 이미 존재하면 업데이트

## 동기화 충돌 정책
- 양쪽 모두 수정된 경우: 사용자에게 선택 UI 제공
- etag 기반 낙관적 동시성 제어
- Calendar: 웹훅 + syncToken (실시간)
- Google Tasks: 폴링 + updatedMin (주기적, 지연 감수)
- Google Tasks는 계정별 updatedMin 커서를 분리 관리한다

## 다중 계정 시 충돌 범위
- 충돌 판정은 Google 계정 단위로 수행한다
- 서로 다른 Google 계정에서 같은 시간대 일정은 "충돌"이 아니라 단순 겹침이다
- 충돌은 동일 계정의 동일 일정이 LifeBase와 Google 양쪽에서 동시 수정된 경우에만 발생한다
- 계정별 syncToken/updatedMin을 독립적으로 관리한다

## Todo 이동/정렬 정책 (Google Tasks 기준)
- Todo 수동 정렬은 동일 계정 + 동일 Task List 내부에서만 허용한다
- 계정 간 또는 Task List 간 이동은 `copy + delete`로 처리한다
- 검색/필터가 켜진 상태에서는 드래그 정렬을 비활성화한다
- 계정 인증 실패 시 해당 계정 Todo는 읽기 전용으로 전환한다
