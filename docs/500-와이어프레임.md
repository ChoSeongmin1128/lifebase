# Design: Wireframes

## 목적
- 주요 플랫폼(Web/Desktop App/Mobile) 화면 뼈대를 빠르게 검증한다.

## 입력
- 캘린더/클라우드/Todo/Gallery/Settings 탭 공통 구조
- 캘린더 전역 계정 필터 + 완료 Todo 표시 토글
- 연간(Year) 뷰 2종 전환(Compact/Timeline)
- 연간 Timeline은 `calendar-year-view.html` 기준 밀도/레이아웃 정렬 필요
- 데스크톱 앱의 폴더 동기화(Nextcloud 유사) UX 필요
- 라이트/다크/시스템 테마 선택 토글
- 동기화 운영 지표는 기본 숨김, 문제 시 alert + 알림 목록 중심 노출
- Cloud는 대시보드보다 파일 브라우저 UX를 우선
- Gallery는 앨범/리스트/상세 3가지 뷰 전환 + 타입 필터/정렬 UX 포함
- Google OAuth 로그인 플로우 UI 별도 프로토타입 필요

## 결정
- Web 와이어프레임: `docs/prototypes/lifebase-web-view.html`
- Desktop App 와이어프레임: `docs/prototypes/lifebase-desktop-app-view.html`
- Mobile 와이어프레임: `docs/prototypes/lifebase-mobile-view.html`
- 캘린더 상세 Year View 실험: `docs/prototypes/calendar-year-view.html`
- 색상 비교/월간 샘플: `docs/prototypes/calendar-palette.html`
- Google 로그인 UI: `docs/prototypes/lifebase-google-login-view.html`
- Cloud 탭 기본 정보는 `동기화 상태/마지막 동기화/해결 필요 여부`로 축약
- `동기화 큐`, `충돌 파일 건수`는 고급 정보로 이동 (기본 화면 미노출)
- 충돌/오류는 alert와 알림 목록에 동시에 기록
- Cloud 메인 뷰는 파일 목록/폴더 탐색으로 고정하고, 특정 폴더 동기화 설정은 Settings > Cloud에서 관리
- Cloud 파일 아이콘은 유형별 인라인 SVG 사용: 문서(file-text), 폴더(folder), 이미지(image), 스프레드시트(grid), PDF(file)
- Cloud 유형 컬럼 제거 (아이콘 + 파일명 확장자로 충분히 구분 가능)
- Cloud 파일 행에 체크박스 + 전체 선택으로 다중 선택 지원, 선택 시 일괄 작업 바(이동/다운로드/삭제) 표시
- Cloud 파일 행 우측에 `⋯` 액션 버튼 → 드롭다운 메뉴(이름 변경/이동/다운로드/삭제)
- Cloud 툴바에 "새 파일" 버튼 추가 (Markdown/텍스트 파일 생성 모달)
- 공유 폴더는 folder 아이콘 + 사람 배지로 구분
- Cloud 빠른 접근 목록 제거 (폴더 네비게이션은 사이드바 서브네비로 통합)
- 캘린더 Year는 `Compact`/`Timeline` 두 모드를 캘린더 툴바 드롭다운에서 전환
- Year Compact는 텍스트 없이 마커 + `+n` 통합 집계 중심으로 구성
- Year Timeline은 좌측 날짜 전용 열 없이 셀 내부 날짜 라벨 중심으로 구성
- 캘린더 화면에는 핵심 탐색만 노출: 뷰 드롭다운(Year·Compact/Year·Timeline/Month/Week) + chevron 이전/다음 + 오늘 + 기간 라벨
- 설정성 옵션(계정 필터, 색상 범례, 완료 Todo 토글)은 Settings 탭으로 이동
- Compact/Timeline 모드는 캘린더 드롭다운에서 직접 전환 (Settings에서 제거)
- 이전/다음 버튼은 chevron 아이콘(◂/▸) 사용, "오늘"은 텍스트 유지
- Web/Desktop 사이드바는 접기/펼치기 토글 지원 (접히면 아이콘만 표시, `localStorage`에 상태 저장)
- 사이드바 아이콘은 인라인 SVG (Feather 스타일) 사용, 유니코드 이모지 사용 금지
- Web/Desktop Cloud 탭은 사이드바 서브네비 방식: Cloud 탭 클릭 시 하위 항목(내 파일/최근 파일/공유됨/중요 표시/휴지통) 펼침, 재클릭 시 토글, 다른 탭 클릭 시 닫힘
- 사이드바 접힌 상태에서 서브네비는 숨김, chevron 아이콘으로 펼침 상태 표시
- 모바일 Cloud는 상단 가로 seg 버튼으로 폴더 필터(내 파일/최근/공유됨/중요/휴지통), 파일 카드에 `⋯` 버튼 → 하단 액션 시트
- 모바일 Cloud 다중 선택: 파일 롱프레스 → 선택 모드 진입, 탭으로 추가 선택/해제, 상단 선택 바에 이동/다운로드/삭제 일괄 액션, ✕로 선택 모드 해제
- Settings 탭은 카테고리별 카드로 구성: 일반(계정/테마) / 캘린더(계정 필터/색상 범례) / Todo(완료 표시) / Cloud(폴더 동기화)
- Web 레이아웃은 뷰포트 높이를 꽉 채우는 flex 레이아웃 사용 (하단 여백 제거)
- 주간(Week) 뷰는 08:00~22:00 1시간 단위 시간 그리드 + 종일 이벤트 영역 + 시간 블록 배치 방식 사용
- 모바일 Year Timeline은 내부 스크롤만 사용 (외부 `.content` 스크롤 차단으로 이중 스크롤 방지)
- 모바일 앱 월간(Month) 뷰는 전체 월 그리드(5~6주)로 렌더링, 이벤트/Todo pill 표시
- 모바일 앱 주간(Week) 뷰는 일별 아젠다 뷰(날짜 헤더 + 이벤트 + Todo 리스트)로 구성
- 모바일 앱 캘린더에 이전/오늘/다음 네비게이션 추가 (chevron 아이콘 + 오늘 텍스트)
- Cloud 파일 테이블 헤더(이름/유형/크기/수정한 날짜)는 드래그로 순서 변경 가능
- Cloud 파일 테이블 컬럼 너비는 헤더 우측 경계 드래그로 조정 가능
- Cloud 파일 테이블 헤더 셀 간 세로 구분선(1px solid border) 표시로 리사이즈 핸들 인지성 확보
- 체크박스/작업 컬럼은 고정 위치, 드래그 대상에서 제외
- Gallery는 격자/리스트/날짜별 3가지 뷰를 아이콘 토글 버튼으로 전환, 기본 뷰는 격자
- Gallery 뷰 구성: 격자(grid) = 순수 썸네일 나열, 리스트(list) = 테이블, 날짜별(detail) = 날짜 헤더로 그룹핑된 썸네일
- Gallery 뷰 전환 아이콘: 격자(grid 4칸) = 격자 뷰, 리스트(bullet list) = 리스트 뷰, 레이아웃(sidebar layout) = 날짜별 뷰
- Web/Desktop은 hover 시 툴팁(격자/리스트/날짜별) 표시, 모바일은 툴팁 없이 아이콘만 사용
- Gallery 썸네일 배지는 확장자 표기 (jpg, mp4, mov 등), IMG/VID 사용 금지
- Gallery 비디오 썸네일에는 반투명 원형 재생 아이콘(▶) 오버레이 표시
- Gallery 리스트 뷰에서 유형 컬럼 제거 (파일명 확장자 + 비디오 재생 아이콘으로 구분)
- Gallery 유형 필터(전체/이미지/비디오)도 아이콘 토글 버튼으로 구성: layers(전체), image(이미지), film(비디오)
- 모바일 Year Compact에서 각 월 그리드 상단에 요일 헤더(일~토) 표시, 날짜가 실제 요일 위치에 정렬
- Cloud 툴바 상단줄: breadcrumb + 리스트/그리드 아이콘 토글(`.view-toggle` 재사용, Gallery와 동일 패턴), "공유 항목" 버튼 제거(사이드바 "공유됨"과 중복)
- Cloud 툴바 하단줄: 검색 + "새로 만들기" 아이콘 버튼(드롭다운 유지) + 업로드 아이콘 버튼 + 정렬 아이콘 드롭다운(이름순/크기순/날짜순 + 오름차순/내림차순)
- Cloud 툴바 액션 버튼(새로 만들기/업로드)은 아이콘 전용 + hover 툴팁 방식, 정렬 아이콘과 시각적 통일
- Cloud 모바일 반응형(760px 이하): 툴바 세로 배치, 검색 전체 너비, 크기 컬럼 숨김
- 모바일 웹(760px 이하): 사이드바 숨기고 하단 고정 탭바(Cloud/Calendar/Todo/Gallery/Settings) 표시
- 모바일 하단 탭바와 사이드바 탭은 상호 동기화
- 모바일 Cloud 탭 상단에 가로 칩 바(내 파일/최근/공유됨/중요/휴지통)로 서브네비 제공, 사이드바 서브네비와 동기화
- 브랜드 로고: `resources/logo_with_bg.svg`(메인 색상 배경) — 헤더 아이콘 + 파비콘 공통 사용
- 로고 리소스: `logo_with_bg.svg`(배경 있음, 파비콘/헤더용), `logo_no_bg.svg`(배경 없음, 스플래시/로딩용)
- Web/Desktop 헤더: 24×24 로고 + 텍스트, 모바일 헤더: 22×22 로고 + 텍스트
- 타이포그래피: Pretendard Variable (CDN dynamic subset) 전 플랫폼 공통 사용
- 오늘 버튼: pill 형태(`.cal-today-btn`, `border-radius: 999px`) 전 플랫폼 통일
- 캘린더 드롭다운 메뉴: 플랫 리스트(Year · Compact / Year · Timeline / 구분선 / Month / Week), 활성 항목 SVG 체크마크
- 모바일 웹(760px 이하): 레이아웃 높이에서 탭바 높이를 차감하여 콘텐츠 겹침 방지 (`calc(100vh - 64px - 56px - env(safe-area-inset-bottom))`)

## 결정 (기술 세부)
- 모바일 Year Timeline의 `table-layout: fixed` 필수 (열 너비 균등 고정)
- Year Timeline의 `border-collapse: separate` + `border-spacing: 0` 사용 (sticky 헤더 호환)
- Year Timeline의 `th`는 `position: sticky; top: 0; z-index: 10` 이상으로 월 헤더 고정
- Year Timeline의 `td`는 `overflow: hidden; max-height: var(--year-row-height)` 로 행 높이 강제 고정
- `syncYearTimelineRowHeight()`는 `wrapper.clientHeight === 0`일 때 계산 스킵 (hidden 상태 보호)
- 탭 전환 시 캘린더 탭 진입하면 `syncYearTimelineRowHeight` 재실행
- Cloud 파일 테이블은 `table-layout: fixed` 사용 (컬럼 너비 고정/리사이즈 지원)
- 컬럼 리사이즈 중 `draggable` 일시 제거로 드래그/리사이즈 충돌 방지
