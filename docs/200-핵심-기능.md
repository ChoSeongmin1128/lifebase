# 핵심 기능

## 목적
- LifeBase에서 처음부터 구현할 전체 기능 범위와 필수 제약을 정의한다.

## 기능 범위
- 클라우드: 로컬 경로 기반 파일 탐색/업로드/다운로드/이동/삭제
- 클라우드 고급: 사용자별 용량 제한, 관리자 용량 설정, 공유 폴더/권한
- 캘린더: 일정 생성/수정/삭제, 5뷰(연간·달력/연간·타임라인/월간/주간/일정)
- Todo: 목록/Todo/완료/우선순위(4단계)/마감일/고정(Pin)/계층(2단계)
- 통합: 일정과 Todo 연결(예: 일정에서 관련 Todo 보기)
- 구글 연동: Google Calendar + Google Tasks 양방향 동기화
- 계정: Google OAuth만 로그인 지원(기본 계정), 추가 Google 계정 동기화 연결
- 계정 필터: 캘린더 전역에서 Google 계정별 일정/Todo 표시 선택
- 갤러리: 미디어 통합 보기, 사진/동영상 재생, 대량 미디어 성능 대응
- 동기화: 모바일 자동 업로드, 데스크톱 로컬 폴더 동기화
- 알림 센터: 동기화/연동 오류, 충돌, 재인증 필요 상태를 누적 기록하고 해결 진입점 제공
- 로그인 UX: Google 계정 선택/권한 동의/연결 완료 플로우 프로토타입 제공

## 동기화 UI 정책
- Cloud 기본 화면은 대시보드가 아니라 파일 브라우저를 우선 표시한다
- 기본 화면에는 `동기화 상태`, `마지막 동기화 시각`, `해결 필요 여부`만 보조 정보로 표시한다
- `동기화 큐`, `충돌 파일 건수` 같은 운영 지표는 기본 숨김이며 고급 정보에서만 확인한다
- 문제 발생 시 상단/인라인 alert로 즉시 노출하고, 동일 항목을 알림 센터 목록에도 남긴다
- 문제 없음 상태에서는 경고/운영 지표를 최소 노출한다

## 갤러리 UI 정책
- 뷰 모드: `격자(기본) / 리스트 / 날짜별` 아이콘 토글 전환
- 격자: 순수 썸네일 그리드, 정렬 기준에 따라 나열
- 리스트: 파일명/날짜 테이블 형식 (유형 컬럼 없음, 확장자+재생 아이콘으로 구분)
- 날짜별: 날짜 헤더로 그룹핑된 썸네일 격자
- 유형 필터: `전체 / 이미지 / 비디오` 아이콘 토글
- 정렬 조건: `수정한 날짜 / 생성한 날짜 / 이름 / 크기` + `오름차순 / 내림차순`
- 비디오 썸네일: 반투명 원형 재생 아이콘(▶) 오버레이
- 썸네일 배지: 파일 확장자 표기 (jpg, mp4, mov 등)

## 갤러리 썸네일 전략
- 서버에서 업로드 시 비동기 워커로 썸네일 생성 (이미지: libvips, 비디오: FFmpeg 프레임 추출)
- 생성 완료 전에는 placeholder 표시

### 썸네일 단계
| 단계 | 크기 | 용도 | 1만 개 기준 |
|---|---|---|---|
| small | 150×150 | 격자, 날짜별, 리스트 (모든 목록형 UI) | ~500MB (50KB×1만) |
| medium | 400×400 | 단일 항목 프리뷰/확대 보기 | 요청 시만 로드 |
| original | 원본 | 다운로드/전체 화면 | 다운로드 시만 |

### 클라이언트 캐시 구조
| 계층 | 저장 위치 | 대상 | 역할 |
|---|---|---|---|
| 1차 | JS 메모리 (Map/Array) | 메타데이터 (~2MB/1만 개) | 뷰 전환, 필터, 정렬 즉시 반영 |
| 2차 | IndexedDB | 메타데이터 | 새로고침/재방문 시 서버 호출 없이 복원 |
| 3차 | 브라우저 디스크 캐시 | 썸네일 이미지 | `Cache-Control`/`ETag` 기반, 뷰 전환 시 재다운로드 없음 |
| 4차 | 서버 API | 메타데이터 + 원본 | syncToken 증분 동기화로 변경분만 수신 |

### 대량 미디어 성능 대응
- 가상 스크롤(virtual scroll): 화면에 보이는 50~100개만 DOM 렌더링
- 썸네일 lazy loading: `IntersectionObserver`로 뷰포트 진입 시 로드
- 뷰 전환: 데이터 레이어(캐시)는 유지, 렌더링만 교체 → API 재호출 없음
- Desktop(Electron): SQLite 로컬 DB를 IndexedDB 대신 사용 가능

## 플랫폼 범위
- 웹: 데스크톱 웹 우선 지원
- 데스크톱 앱: macOS + Windows 둘 다 지원
- 모바일 앱: iOS + Android 둘 다 지원

## 구현 제약
- 인증은 Google OAuth + JWT 방식(로컬 비밀번호 로그인 없음)
- 사용자 간 정보 완전 격리(다른 사용자의 존재/정보를 알 수 없어야 함)
- 기능 규칙은 서버에서 단일 관리(플랫폼별 규칙 분기 금지)
- 공유 링크는 10분 유효 토큰 + 수락 후 ACL 권한 모델

## 결정 (Cloud 정렬)
- 기본 정렬: 이름순 오름차순
- 정렬 옵션: 이름순 / 크기순 / 수정한 날짜순 / 생성한 날짜순
- 폴더 항상 우선(Folders-first): 정렬 기준/방향에 관계없이 폴더가 파일 위에 배치
- 크기순일 때 폴더는 이름순 정렬 (폴더에 크기 표시 없음)
- 방향 토글: 각 기준에 독립적 기본 방향, 기준 변경 시 기본 방향으로 리셋
- 테이블 헤더 클릭 정렬: 클릭 시 해당 컬럼 기준 정렬, 재클릭 시 방향 반전, 활성 컬럼에 SVG ▲/▼ 아이콘
- Tie-breaker: 이름 오름차순 (같은 폴더 내 파일명 유일)
- 폴더별 개별 정렬 미지원 (0.x 단계), Cloud 전체 하나의 정렬 상태

## 결정 (Gallery 정렬)
- 기본 정렬: 수정한 날짜 내림차순
- 정렬 옵션: 수정한 날짜 / 생성한 날짜 / 이름 / 크기
- 뷰별 적용: 격자(좌→우, 위→아래) / 리스트(테이블 행 순서) / 날짜별(날짜 그룹 내 정렬)
- 날짜별 뷰: 현재 정렬 기준의 날짜 필드로 그루핑, 이름/크기순일 때는 수정 날짜로 그루핑
- Tie-breaker: 수정한 날짜 내림 → 이름 오름

## 결정 (정렬 상태 영속성)
- 프로토타입/0.x: `localStorage` 단독 (`lifebase-sort-{viewName}`)
- 1.0 이후: 서버 저장 추가, 로그인 시 서버→localStorage 동기화
- 캘린더 뷰는 시스템 고정 정렬이므로 저장 대상 아님
- "기본 정렬로 돌아가기" 옵션을 정렬 드롭다운 하단에 배치
