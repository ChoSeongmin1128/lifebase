# 캘린더 기능

## 목적
- LifeBase 캘린더의 기능 범위를 정의한다.

## 상태
- 연간(Year) 뷰 표시 규칙 1차 확정 (2026-02-24)
- 연간(Year) 뷰 이벤트/Todo 분리 표시 + 완료 Todo 토글 규칙 반영 (2026-02-24)
- 캘린더 전역 계정 필터(뷰 공통) 규칙 반영 (2026-02-24)
- 월/주 뷰 1차 프로토타입 개선 반영 (2026-02-24)
- 연간(Year) 2종 뷰(Compact/Timeline) 전환 프로토타입 반영 (2026-02-24)
- 연간 Timeline 기준 렌더(동적 행 높이/세로 스크롤 제거) 반영 (2026-02-24)
- 연간 Compact 마커 전용(텍스트 제거) 규칙 반영 (2026-02-24)
- 프로토타입 3차: 캘린더 UI 설정 분리, 주간 뷰 재설계 반영 (2026-02-24)
- 프로토타입 4차: 드롭다운 플랫 리스트화, chevron 네비게이션, 오늘 pill 버튼, Pretendard 폰트 반영 (2026-02-24)
- 프로토타입 5차: 캘린더 UX 개선 — 드롭다운 SVG+한글, Agenda 빈 날짜 숨김, Week 시간 설정, Month/Week 스패닝 바, 이벤트 시간 범위 표시, 겹침 분할 배치, 3플랫폼 동기화 (2026-02-25)

## 기능 범위 (초안)
- 일정 CRUD (제목, 시간, 장소, 메모)
- 6뷰 전환: 연간·달력(Compact) / 연간·타임라인(Timeline) / 월간 / 주간 / 3일 / 일정(Agenda)
- 종일 일정 / 시간 지정 일정 구분
- 반복 일정 (매일/매주/매월/매년 + 커스텀 RRULE)
- 반복 일정 수정 (이번만/이후 전체/전체)
- 알림 설정 (다중 알림, 팝업/이메일)
- 다중 캘린더 (업무/개인 등 그룹 분리, 표시/숨김 토글)
- 캘린더별 색상, 이벤트별 색상 오버라이드
- Google Calendar 양방향 동기화
- Todo 마감일 캘린더 표시 (Google Tasks 연동)
- 다중 Google 계정 일정 통합 표시 + 계정별 표시/숨김 선택

## 결정 (캘린더 공통 + Year View 프로토타입 1차)
- 기간 일정 바는 lane 고정 방식으로 렌더링한다. (날짜가 바뀌어도 같은 lane 유지)
- 기간 일정 바는 셀당 최대 3개까지 노출한다.
- 텍스트는 셀당 1개만 노출한다.
- 초과 일정 집계는 `+n` 형식으로 이벤트와 Todo를 통합 표시한다.
- `+n`은 숨겨진 기간 일정 + 숨겨진 단일 일정 + 숨겨진 Todo 마감일의 총 개수로 계산한다.
- 텍스트 우선순위는 공휴일 > 시작일(또는 단일) 기간 일정 > 단일 일정 순서를 따른다.
- Todo는 Google Tasks의 due(날짜) 기반으로 Year 뷰에 표시하며, 완료 Todo 기본값은 숨김 처리한다.
- Todo는 이벤트와 시각적으로 분리해서 칩 형태로 표시한다. (세로 바 미사용)
- 사용자가 완료 Todo 표시 여부를 토글할 수 있어야 한다. (기본: 활성 Todo만 표시)
- 완료 Todo 토글은 Settings > Todo에서 설정한다. (캘린더 화면 미노출)
- 사용자가 표시할 Google 계정을 선택할 수 있어야 한다. (기본: 전체 계정)
- 계정 필터는 Settings > 캘린더에서 설정한다. (캘린더 화면 미노출)
- 계정 선택 상태는 월/주/일/연간 뷰에 공통 적용해야 한다.
- 계정 필터는 이벤트와 Todo 모두에 동일하게 적용한다.
- 계정별 식별 정보(예: 계정 이메일/라벨)는 툴팁 또는 상세 보기에서 확인 가능해야 한다.
- 연간(Year) 뷰는 `Compact`(월 카드 요약형)과 `Timeline`(날짜 고정 매트릭스형) 2가지를 제공해야 한다.
- Compact/Timeline 전환은 캘린더 화면 툴바 드롭다운에서 직접 선택한다.
- 연간 2종 뷰 모두 `lane 최대 3`, `+n` 통합 집계 규칙을 공통으로 적용한다.
- 텍스트 슬롯(`텍스트 1`) 규칙은 Timeline에만 적용한다.
- Year `Compact`는 스캔 중심 뷰로 마커(이벤트/단일/Todo)와 `+n`만 노출하고 텍스트는 노출하지 않는다.
- Year `Timeline`은 좌측 별도 날짜 열 없이 셀 내부 날짜 라벨만 사용한다. (요일은 색상/툴팁으로 보조)
- Year `Timeline`은 hover/tooltip을 통해 날짜 컨텍스트를 보완한다.
- 데스크톱 Year 뷰는 내부 세로 스크롤 없이 동작해야 하며, 행 높이는 뷰 높이에 맞춰 자동 계산한다.
- 모바일 Year 뷰는 가로 스크롤을 유지하고, 기간 일정 바/텍스트/Todo/+n 렌더 규칙은 데스크톱과 동일하게 적용한다.

## 결정 (캘린더 화면 UI)
- 캘린더 화면 툴바: 단일 드롭다운(Year·Compact / Year·Timeline / Month / Week / Agenda) + chevron 이전/다음 + 오늘 + 기간 라벨
- 설정성 옵션(계정 필터, 색상 범례, 완료 Todo 토글)은 Settings 탭으로 이동
- Compact/Timeline 모드는 캘린더 드롭다운에서 직접 전환 (Settings에서 제거)
- 색상 범례는 Settings > 캘린더에서 확인 가능
- Month 뷰는 이벤트/Todo를 통합 리스트로 표시하고, Todo는 체크 SVG 아이콘으로 구분한다.
- Month 뷰 셀 클릭 동작은 항목 타입별로 분기한다: 이벤트는 일정 상세/편집, Todo는 Todo 상세(완료 토글 + 편집)로 진입한다.

## 결정 (3일 뷰)
- 드롭다운에 별도 뷰 항목으로 추가 (Week과 독립)
- 시간 그리드 구조는 Week 뷰와 동일 (종일 영역 + 시간 블록 + 겹침 처리)
- 3일 단위로 chevron 이전/다음 이동
- Quick Create, 이벤트 클릭 상세, 드래그 이동 등 Week 뷰와 동일한 CRUD 지원
- 모바일에서 특히 유용: 7일 Week은 열이 좁아 가독성 저하, 3일이면 충분한 이벤트 표시 공간 확보

## 결정 (주간 뷰)
- 시간 그리드: 기본 08:00~22:00, Settings에서 0~24시 범위로 변경 가능
- 시작 시간 30분 전부터 약한 구분선(dashed)으로 표시 (라벨 없음)
- 종일 이벤트 영역: 그리드 상단에 별도 영역으로 분리
- 시간 지정 이벤트: 시작 시간/기간 기반으로 그리드 내 절대 위치 배치
- 이벤트 블록에 시간 범위 표시 (예: 09:00-10:00)
- 겹치는 시간 이벤트: 같은 시간대를 나란히 열 분할 배치 (Google Calendar 방식)
- 기간 이벤트(다일간): 종일 이벤트 영역에 grid-column 기반 스패닝 바로 표시
- Todo: 종일 이벤트 영역에 체크마크 칩 형태로 표시
- 오늘 날짜: 헤더에 원형 배경 강조
- 요일 헤더: 요일 약어 + 날짜 숫자 표시

## 결정 (월간 뷰 스패닝 바)
- 기간 이벤트는 주 단위 행에서 연속 바(spanning bar)로 렌더링
- 주 경계에서 잘리며 시작/중간/끝 모서리 스타일 구분
- 이어지는 이벤트도 항상 제목 표시
- 레인 배정(assignLanes)으로 겹치지 않게 배치

## 결정 (Agenda 뷰)
- 일정 있는 날짜만 표시 (빈 날짜 숨김)
- 전체 범위에 일정 없으면 빈 상태 메시지 표시
- 무한 스크롤 양방향 지원

## 결정 (캘린더 정렬 — 시스템 고정)
- 사용자 정렬 옵션 없음 (모든 캘린더 뷰는 시스템 고정 규칙)
- Lane 배정 순서: 시작일 오름차순 → 기간 내림차순(긴 이벤트 우선) → 제목 가나다순
- Year/Month/Week 종일 영역: 동일 lane 배정 규칙 적용
- Week 시간 그리드: 시간 위치(Y축) 배치, 겹치는 이벤트는 시작 시간 오름 → 종료 시간 오름으로 열 배정
- Agenda 날짜 내: 종일 이벤트 → 시간 이벤트(시작 시간 오름) → Todo(우선순위 내림 → 제목 가나다)
- 셀 내 단일 항목: 종일 이벤트 → 시간 지정 이벤트(시작 시간 오름) → Todo

## 결정 (이벤트 CRUD)
- 빠른 생성: 빈 슬롯 클릭 → Quick Create 팝오버(제목+시간만) → 필요 시 "상세" 버튼으로 풀 모달 확장
- 상세 모달 필드: 제목, 시간(시작/종료), 종일 토글, 장소, 메모, 캘린더 선택, 색상, 반복, 알림
- 수정: 이벤트 클릭 → 상세 팝오버 → "수정" 버튼으로 상세 모달 진입
- 삭제: Undo 토스트 5초 (하단에 "삭제됨 — 실행 취소" 표시, 클릭으로 복구)
- ETag 기반 낙관적 동시성 제어 (충돌 시 사용자 선택 UI)
- 드래그로 시간/날짜 변경: Week/3일/Month에서 지원 (1.x)

## 결정 (뷰별 CRUD 지원 범위)
- Year Compact: 생성 X, 날짜 클릭 → 해당 날짜 Agenda로 이동, 드래그 X
- Year Timeline: 생성 X, 이벤트 클릭 → 상세 팝오버(수정/삭제), 드래그 X
- Month: 날짜 셀 클릭 → Quick Create, 이벤트 클릭 → 상세 팝오버, 드래그로 날짜 이동 (1.x)
- Week: 빈 슬롯 클릭 → Quick Create, 이벤트 클릭 → 상세 팝오버, 드래그로 시간/날짜 이동 (1.x)
- 3일: Week과 동일
- Agenda: 상단 + 버튼 → Quick Create, 이벤트 클릭 → 상세 팝오버, 드래그 X
- 공통: 상세 팝오버는 모든 뷰에서 동일한 컴포넌트 사용 (제목/시간/캘린더/메모 표시 + 수정/삭제 버튼)

## 결정 (반복 일정)
- RRULE (RFC 5545) 기반: FREQ, INTERVAL, BYDAY, BYMONTHDAY, UNTIL/COUNT 지원
- UI: 반복 빌더 (매일/매주/매월/매년 + 커스텀 간격 + 요일 선택 + 종료 조건)
- 수정 시 3-way 선택: "이 일정만" / "이후 모든 일정" / "모든 일정"
- 삭제 시 3-way 선택: 동일 (단일 삭제는 Undo 토스트, 전체 삭제는 확인 다이얼로그)
- "이 일정만" 수정: 해당 인스턴스 PATCH → 예외 생성 + EXDATE 추가
- "이후 모든 일정" 수정: 원본 RRULE에 UNTIL 추가 + 새 반복 이벤트 생성
- "모든 일정" 수정: 부모 이벤트 직접 수정
- Google Calendar recurrence 필드 양방향 동기화

## 결정 (알림)
- 채널: Push(FCM/APNs) + In-App(토스트/배지), Email은 1.x에서 추가
- 플랫폼별 Push: Web(FCM Web Push API), Desktop macOS(Electron 알림), Desktop Windows(Electron 알림), iOS(APNs), Android(FCM)
- In-App: 토스트(일시적) + 알림 센터 배지(누적) — 5플랫폼 공통
- 이벤트 알림: Google Calendar reminders.overrides 필드 동기화 (method: popup, minutes)
- 기본 알림: 캘린더별 defaultReminders 설정 상속, 이벤트별 커스텀 오버라이드
- 최대 5개 알림 (Google Calendar 제한과 동일)
- 방해 금지(DND): 시간대 스케줄 설정 + "긴급만 허용" 옵션, Settings에서 관리

## 결정 (참석자)
- Apple 캘린더 수준의 참석자 초대/응답 관리 포함
- 이벤트 생성/수정 시 이메일로 참석자 추가
- 참석 상태 표시 (수락/거절/미응답)
- Google Calendar API attendees[] 필드 양방향 동기화
